---- Casos Borde PC3 - Trapping Rain Water II ----

--- Ejecucion ---
./build/unit_tests.exe

--- Caso Borde 1: Grilla minima 3x3 ---
Input:
  3 3 3
  3 1 3
  3 3 3

Output: 2

Analisis: Es el tamano minimo posible para retener agua. Solo existe
una celda interior de altura 1 rodeada por bordes de altura 3.
El agua retenida es 3-1 = 2. Cualquier grilla menor no puede
retener agua porque no tiene celdas interiores.

--- Caso Borde 2: Sin agua acumulable ---
Input:
  1 1 1
  1 2 1
  1 1 1

Output: 0

Analisis: La celda interior tiene altura 2, mayor que los bordes de
altura 1. El agua no puede quedar atrapada porque fluye libremente
hacia los bordes que son mas bajos. Una solucion superficial podria
intentar calcular agua negativa, pero el algoritmo lo maneja
correctamente retornando 0.

--- Caso Borde 3: Todas las celdas con la misma altura ---
Input:
  2 2 2 2
  2 2 2 2
  2 2 2 2
  2 2 2 2

Output: 0

Analisis: Cuando no hay diferencia de alturas entre celdas interiores
y bordes no puede haber agua atrapada. Este caso prueba que el
algoritmo maneja correctamente los empates de altura sin calcular
agua incorrecta.

--- Caso Borde 4: Grilla menor a 3x3 ---
Input:
  3 3
  3 3

Output: 0

Analisis: Una grilla 2x2 no tiene celdas interiores, por lo tanto
es imposible retener agua. El algoritmo detecta este caso con la
condicion if (filas < 3 || cols < 3) y retorna 0 inmediatamente
sin procesar nada.

--- Caso Borde 5: Grilla de una sola fila ---
Input:
  3 3 3 3 3

Output: 0

Analisis: Una grilla de una sola fila no tiene dimension vertical,
por lo tanto todas las celdas son bordes y ninguna puede retener
agua. Al igual que el caso anterior el algoritmo lo detecta con
la condicion de tamano minimo y retorna 0.
